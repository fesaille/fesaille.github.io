Star/snowflake schemas

The star schema separates business process data into facts, which hold the measurable, quantitative data about a business, and dimensions which are descriptive attributes related to fact data: - Fact tables record measurements or metrics for a specific event, generally consist of numeric values, and foreign keys to dimensional data where descriptive information is kept - Dimension tables usually have a relatively small number of records compared to fact tables, but each record may have a very large number of attributes to describe the fact data.

"Snowflaking" is a method of normalizing the dimension tables in a star schema.
